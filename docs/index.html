Configuration

Add to your postal.yml. The only required field is oidc.issuer — Postal fetches public keys automatically.

api:
  enabled: true
  required_azp: my-service-account   # optional — restrict to one KC/Entra/Google client

oidc:
  issuer: https://login.example.com/realms/myrealm   # must match the 'iss' claim in your JWTs

scim:
  enabled: true
  bearer_token: <scim-token>

Authentication

Postal acts as a Resource Server — it never issues tokens itself. Your identity provider (Keycloak, Microsoft Entra, or Google) issues a JWT via the client_credentials grant. Postal verifies it using the provider's public keys, fetched automatically from {issuer}/.well-known/openid-configuration and cached for 1 hour. No secret is stored in Postal.

Every API request must include:

Authorization: Bearer <access_token>

Keycloak

1. Create a service-account client

In your realm → Clients → Create client:

FieldValue
Client IDpostal-api (or any name)
Client authenticationON
Authentication flowService accounts roles only

Copy the Client secret from the Credentials tab.

2. Get a token

curl -s -X POST \
  https://login.example.com/realms/myrealm/protocol/openid-connect/token \
  -d grant_type=client_credentials \
  -d client_id=postal-api \
  -d client_secret=<secret> \
  | jq -r .access_token

3. Configure Postal

oidc:
  issuer: https://login.example.com/realms/myrealm

api:
  enabled: true
  required_azp: postal-api   # the client_id above

The JWKS URI is auto-discovered. No other config needed.

Microsoft Entra (Azure AD)

1. Register an application

Azure Portal → App registrations → New registration. Then:

  • Certificates & secrets → New client secret → copy the value
  • Expose an API → Add a scope (e.g. postal.admin) — or use the default .default scope

Note your Application (client) ID and Directory (tenant) ID.

2. Get a token

curl -s -X POST \
  https://login.microsoftonline.com/<tenant_id>/oauth2/v2.0/token \
  -d grant_type=client_credentials \
  -d client_id=<app_id> \
  -d client_secret=<secret> \
  -d scope=<app_id>/.default \
  | jq -r .access_token

3. Configure Postal

oidc:
  issuer: https://login.microsoftonline.com/<tenant_id>/v2.0

api:
  enabled: true
  required_azp: <app_id>   # Application (client) ID

Postal will auto-discover JWKS from https://login.microsoftonline.com/<tenant_id>/v2.0/.well-known/openid-configuration.

Google

1. Create a service account

Google Cloud Console → IAM & Admin → Service Accounts → Create. Then:

  • Go to the service account → Keys → Add key → JSON → download
  • Note the client_email and client_id from the JSON file

2. Get a token (using the Google auth library or oauth2l)

# with oauth2l (https://github.com/google/oauth2l)
oauth2l fetch --credentials service-account.json --scope openid email

# or with Python google-auth
python3 - <<'EOF'
import google.auth, google.auth.transport.requests
creds, _ = google.auth.default(scopes=["openid","email"])
creds.refresh(google.auth.transport.requests.Request())
print(creds.token)
EOF

3. Configure Postal

oidc:
  issuer: https://accounts.google.com
  jwks_uri: https://www.googleapis.com/oauth2/v3/certs   # optional, auto-discovered

api:
  enabled: true
  required_azp: <client_id from service-account.json>

API v2 reference

All requests require Authorization: Bearer <token>. Standard HTTP status codes: 200 201 204 401 404 422.

POST/api/v2/organizationsCreate (auto-provisions SMTP server)
GET/api/v2/organizations/:permalinkGet
PATCH/api/v2/organizations/:permalinkUpdate
DELETE/api/v2/organizations/:permalinkDelete
