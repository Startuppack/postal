Open-source mail server — extended by Startup Pack with a full admin REST API, member roles, and auto-provisioned SMTP servers.

API reference View on GitHub
This is an unofficial fork. Maintained by Startup Pack, not affiliated with the official Postal project (Krystal Hosting Ltd). The API v2 and role system were created by Startup Pack.

What this fork adds

🔌

Admin REST API v2

Full provisioning API for orgs, servers, domains, credentials and users — no Rails console needed.

👥

Multi-user orgs

Add multiple users to an organization with granular roles via API or SCIM.

🔒

Role-based access

admin, member, readonly — enforced across all web UI controllers.

⚡

Auto SMTP server

Creating an org via API or SCIM automatically provisions a Live SMTP server and default credential.

🔐

SCIM v2 + OIDC

SCIM user/group sync and JIT OIDC user provisioning on first SSO login.

🛠

Upstream fixes

UTF-8 subject encoding, SMTP timeouts, stale webhook locks, X-Envelope-From, Message-ID idempotency.

Configuration

Add to your postal.yml:

api:
  enabled: true
  bearer_token: <your-secret-token>

scim:
  enabled: true
  bearer_token: <scim-token>

API v2 reference

All requests require Authorization: Bearer <token>. Standard HTTP status codes: 200 201 204 401 404 422.

Organizations

MethodPathDescription
GET/api/v2/organizationsList all
POST/api/v2/organizationsCreate (auto-provisions SMTP server)
GET/api/v2/organizations/:permalinkGet
PATCH/api/v2/organizations/:permalinkUpdate
DELETE/api/v2/organizations/:permalinkDelete
POST/api/v2/organizations/:permalink/suspendSuspend / unsuspend
{ "name": "Acme", "permalink": "acme", "time_zone": "UTC", "owner_email": "admin@acme.com" }

Members

MethodPathDescription
GET/api/v2/organizations/:org/membersList members
POST/api/v2/organizations/:org/membersAdd user to org
PATCH/api/v2/organizations/:org/members/:userChange role
DELETE/api/v2/organizations/:org/members/:userRemove member
{ "user_id": "user@example.com", "role": "member" }

Servers

MethodPathDescription
GET / POST/api/v2/organizations/:org/serversList / create
GET / PATCH / DELETE/api/v2/organizations/:org/servers/:serverGet / update / delete
POST…/:server/suspendSuspend / unsuspend

Domains

MethodPathDescription
GET / POST…/:server/domainsList / add (returns DKIM record)
GET / DELETE…/:server/domains/:uuidGet (DKIM + DNS status) / delete
POST…/domains/:uuid/verifyForce-mark verified
POST…/domains/:uuid/dns_checkRe-check DNS records

Credentials

MethodPathDescription
GET / POST…/:server/credentialsList / create (returns generated key)
GET / DELETE…/:server/credentials/:uuidGet / delete

Users

MethodPathDescription
GET / POST/api/v2/usersList / create
GET / PATCH / DELETE/api/v2/users/:idGet / update / delete (by UUID or email)

All list endpoints accept ?page=1&per_page=50 (max 200). Errors: {"errors": ["Name can't be blank"]}.
